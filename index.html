Why Simple Docs?

Ever thought "I should write this down" while working and end up not doing it? I did.

Though robust, all the documentation software I used required too many steps between initial thought and execution. By the time I opened a new browser tab, remembered my login information, navigated the admin panel and created a new document page I had lost my current mental context and any sense of flow.

With Simple Docs I can stay on the command-line, use my preferred text editor, write notes in Markdown, organize everything using files and directories, and have it all mirrored online. No compilation needed. No caches to clear. No context switches. No more long series of steps. Setup once and get back to writing docs.

Simple docs doesn't replace the many robust and powerful professional documentation software. Instead it makes it easier to capture, organize, and edit knowledge without all the steps between thought and execution.

Setup

The only requirements for Simple Docs is Python +2.6, Python PIP and a web server. Setup is straight-forward:

$ git clone https://github.com/chrislaskey/simple-docs project_directory
$ cd project_directory
$ ./setup.sh
$ ln -s your_docs_directory ./docs

The last step is setting up the webserver. Simple Docs uses Python Flask under the hood any of the Flask web server deployment options will work.

FAQ

What are some of the use cases for Simple Docs?
Simple Docs is great for capturing all the information that usually falls between the cracks. At work I use it to collaboratively keep notes about our evolving systems architecture. Personally I use it to organize programming, systems, and design notes from over the years.
What platforms can Simple Docs run on?
The application requires Python +2.6, Python PIP and a web server. The search functionality requires grep, ls and find. The application has been tested in Debian and RHEL Linux distributions. Though not tested, there should be nothing to prevent it from running on Mac OS X, Solaris or FreeBSD.
How complex is Simple Docs under the hood?
Simple Docs extends the idea of simplicity to the technical stack. Instead of creating menus and content hiearchies, what the file system is directly mirrored. Instead of creating separate website users and permissions, existing Unix user and group permissions are used. Search online using the same tools as the command line.
What's the recommended web server?
The best advice is to use what is familiar. Apache + Mod_WSGI is the most common, though Nginx + uWSGI and Nginx + gunicorn are becoming increasingly popular python web stacks.
Are symlinks allowed for the ./docs directory?
Not only are they allowed, they are encouraged as standard practice. This allows docs to remain in their current fixed location, or fixed locations.
How do I restrict viewing access?
A core tenant of Simple Docs is to rely on systems that already exist. Permissions can be handled using ip filtering or web server authorization. The former can be handled directly with iptables rules. Most web servers support both ip filtering and simple authorization schemes.
How do I manage write access?
A core tenant of Simple Docs is to rely on systems that already exist. Simple Docs relies on file system user/group file permissions.
How does the weighted search work?
Search terms are filtered into individual search words, composed of only letters and numbers. Each term is then searched against filenames and file contents. Filename matches carry more weight than a file content match.
